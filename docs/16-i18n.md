# 16. Internationalization (i18n)

Render your screens in the user's language using standard Spring `MessageSource` plus the
locale from their browser.

## Step 1 — message bundles

`src/main/resources/messages.properties` (default):

```properties
greeting=Hello, {0}!
items.count=You have {0} items
```

`src/main/resources/messages_es.properties`:

```properties
greeting=¡Hola, {0}!
items.count=Tienes {0} elementos
```

Spring Boot auto-configures a `MessageSource` from these.

## Step 2 — use the locale in a screen

The client's locale comes from the `Accept-Language` header. Capture it on mount and use it
when resolving messages:

```kotlin
@LiveComponent("Home")
@Route("/")
class HomeScreen(private val messages: MessageSource) : ServerComponent, LiveLifecycle {

    private var locale: Locale = Locale.ENGLISH

    override fun onMount() {
        locale = LiveContext.current().locale()      // from Accept-Language
    }

    private fun t(code: String, vararg args: Any) =
        messages.getMessage(code, args, locale)

    override fun render(): UiNode =
        div(cls("card"),
            h1(t("greeting", "Kumar")),
            p(t("items.count", 3)))
}
```

A Spanish browser sees "¡Hola, Kumar!"; everyone else gets the default.

> Why `onMount`? `LiveContext.current()` is available in `onMount` and actions. Capturing
> the locale once keeps `render()` simple and pure.

## Switching language manually

Store a chosen locale in `@LiveState` and let an action change it:

```kotlin
@LiveState var lang = "en"
@LiveAction fun setLang(code: String) { lang = code }
// in render: messages.getMessage(code, args, Locale.forLanguageTag(lang))
```
